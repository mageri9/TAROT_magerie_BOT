# 🌙 Tarot Magerie Bot 🌙
# 🎮 Живой бот: [@MyTarotBotMyTarotBot](https://t.me/MyTarotBotMyTarotBot)

Асинхронный Telegram-бот для гадания на Таро с AI-Оракулом, персонализацией и отказоустойчивостью.

![Image](https://github.com/user-attachments/assets/248952eb-f1be-41af-aa92-014a3e3a647a)


## ✨ Возможности

### 👤 Для пользователя
- 🎴 Полная колода (78 карт) с прямыми и перевёрнутыми толкованиями
- 🔮 Карта дня (1 раз/24ч) + кастомные расклады
- 💬 **AI-Оракул**: ответы на вопросы с учётом вашей истории и контекста
- 📝 Ввод настроения/вопроса перед гаданием для точной интерпретации
- 📊 Личный профиль: статистика
- ⚡ Мгновенные ответы благодаря умному кэшированию

### ⚙️ 
- 🛡️ `CircuitBreaker` для AI: авто-фолбэк и защита при сбоях LLM API
- 💾 Redis-кеш ответов Оракула с настраиваемым TTL
- 🔄 Асинхронные бэкапы `pg_dump` с Redis-локом от параллельного запуска
- 🔌 Singleton-клиент Redis, упрощённая DI и тестирование
- 🐳 Полный `docker-compose` стек: бот + Postgres + Redis
- 🧪 40+ `pytest`-тестов, линтеры, CI/CD через GitHub Actions
- 📈 Sentry — отслеживание ошибок в реальном времени

## 🏗️ Стек и архитектура

| Компонент | Технология |
|-----------|------------|
| Язык | Python 3.12 |
| Бот | `aiogram 3.x` (асинхронный, FSM) |
| БД | `PostgreSQL` + `alembic` (миграции) |
| Кеш / Очереди | `Redis` |
| AI / RAG | Gemma 4 (AITUNNEL API) + контекст пользователя |
| Деплой | `Docker Compose` + GitHub Actions |
| Логирование | `loguru` |
| Тесты | `pytest` + `pytest-asyncio` |

```
src/
├── core/          # конфиг, БД, Redis, логгер, DI
├── crud/          # операции с данными (SQLAlchemy)
├── handlers/      # user + admin роутеры
├── middlewares/   # rate limit, логирование, error handler
├── services/      # AI, Tarot, карта дня, кеш, бэкапы
├── keyboards/     # inline + reply клавиатуры
├── filters/       # IsAdmin, ChatType, кастомные
└── utils/         # хелперы, валидаторы
```
🛠️ Админ-панель (/admin)

    📸 Получение file_id для фото
    🃏 Управление рубашками карт
    📊 Статистика пользователей
    📦 Ручной запуск бэкапа
    🧹 Очистка AI-кэша
    🔄 Сброс лимита карты дня для пользователя
    📜 Просмотр логов ошибок

🧠 RAG-логика

    156 толкований в БД (78 карт × 2 положения)
    Промпт: [карта] + [положение] + [БД-значение] + [контекст пользователя] + [история]
    Fallback-модель при сбое основной
    Изоляция контекста: разные запросы → разные кеш-ключи

📈 Мониторинг

    Sentry: трек ошибок с контекстом
    Логирование: loguru, ротация по размеру/времени, уровни DEBUG/INFO/ERROR
    Админка: статус БД/Redis/AI, счётчики ошибок

🔐 Безопасность

    Pydantic-валидация конфига при старте
    Параметризованные SQL-запросы (защита от инъекций)
    Redis Rate Limiter на входящие сообщения
    Фильтр IsAdmin + проверка ADMIN_IDS для админ-команд
    Секреты только через .env / GitHub Secrets

## 🚀 Быстрый старт

1. **Клонируйте и перейдите в папку:**
   ```bash
   git clone https://github.com/mageri9/TAROT_magerie_BOT.git
   cd TAROT_magerie_BOT
   ```

2. **Настройте окружение:**
   ```bash
   cp .env.example .env
   # Отредактируйте .env (см. таблицу ниже)
   ```

3. **Запустите стек:**
   ```bash
   docker compose up -d --build
   ```
   Бот готов. Логи: `docker compose logs -f bot`

## ⚙️ Конфигурация (`.env`)

| Переменная | Описание | Пример / По умолчанию |
|------------|----------|------------------------|
| `BOT_TOKEN` | Токен Telegram Bot | `123456:ABC-...` |
| `DATABASE_URL` | Async-строка подключения | `postgresql+asyncpg://user:pass@db:5432/tarot` |
| `REDIS_URL` | Строка подключения | `redis://redis:6379/0` |
| `AI_API_KEY` | Ключ LLM-провайдера | `sk-...` |
| `AI_MODEL` | Основная модель | `gemma-4-12b-it` |
| `AI_CIRCUIT_FAILURE_THRESHOLD` | Ошибок до срабатывания Circuit Breaker | `5` |
| `AI_CIRCUIT_RESET_TIMEOUT` | Секунд до автоматического сброса цепи | `300` |
| `AI_CACHE_ENABLED` | Включить кеш AI-ответов | `true` |
| `AI_CACHE_TTL` | Время жизни кеша (сек) | `3600` |

> 💡 Всегда сверяйтесь с `.env.example` — там актуальный список всех ключей.


## 🛡️ Надёжность и эксплуатация

| Механизм | Как работает |
|----------|--------------|
| **Circuit Breaker** | При `N` подряд ошибках запросы к AI блокируются, бот отдаёт заготовленный ответ или фолбэк-модель. Восстанавливается автоматически. |
| **Кеширование** | Повторяющиеся запросы к Оракулу отдаются из Redis без нагрузки на LLM. TTL настраивается. |
| **Бэкапы** | Асинхронный `pg_dump` с Redis-локом. Статус и кнопка очистки кеша доступны в админ-панели. |
| **Rate Limiting** | Защита от флуда на уровне Redis. |
| **Healthcheck** | Встроен в `docker-compose.yml` (проверка БД, Redis, API). |
